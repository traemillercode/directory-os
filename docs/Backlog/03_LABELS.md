# GitHub Labels

Create the following labels in the repository before creating issues. Colors are suggestions and can be adjusted to match repository conventions.

## Type

| Label | Description |
|---|---|
| `type:epic` | Tracking issue grouping related child issues |
| `type:feature` | New user-facing functionality |
| `type:task` | Discrete engineering task within a feature/epic |
| `type:bug` | Defect in existing functionality |
| `type:docs` | Documentation-only work |
| `type:chore` | Maintenance, tooling, or non-functional work |
| `type:spike` | Time-boxed research/investigation |

## Phase / Milestone

| Label | Description |
|---|---|
| `phase:mvp` | Required for MVP (Phase 1, weeks 1–12) |
| `phase:2` | Phase 2 (Enhanced Automation, weeks 13–24) |
| `phase:3` | Phase 3 (Scale & Intelligence, weeks 25–52) |

## Area

| Label | Description |
|---|---|
| `area:auth` | Authentication & authorization |
| `area:database` | Database schema, migrations, RLS |
| `area:listings` | Listing CRUD & dashboard |
| `area:import-export` | Bulk CSV import/export |
| `area:ai` | AI/Claude/Gemini integration |
| `area:analytics` | Analytics & reporting |
| `area:leads` | Lead capture & management |
| `area:infra` | Deployment, CI/CD, hosting |
| `area:security` | Security & compliance |
| `area:testing` | Test automation & QA |
| `area:docs` | Product/engineering documentation |

## Priority

| Label | Description |
|---|---|
| `priority:p0-blocker` | Must be done before dependent work can start |
| `priority:p1-high` | Required for MVP launch |
| `priority:p2-medium` | Important but not launch-blocking |
| `priority:p3-low` | Nice-to-have / backlog |

## Status

| Label | Description |
|---|---|
| `status:blocked` | Cannot proceed until a dependency is resolved |
| `status:needs-design` | Requires design/spec before implementation |
| `status:ready` | Ready to be picked up |

## Special

| Label | Description |
|---|---|
| `idea` | Community/team feature idea per `docs/Product/04_IDEAS.md` submission process |
